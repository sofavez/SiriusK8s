Практика 1 — vim, mc, asciinema и анализ nginx-логов
### Задача
Мне нужно разобраться, как работают инструменты **vim**, **mc** и **asciinema**,
потренироваться на подготовленных файлах, записать всю работу на видео через
asciinema, а затем решить практическую задачу: найти 10 IP-адресов
с наибольшим количеством запросов в файле `/var/log/nginx/access.log` и дополнительно сделать то же самое для JSON-лога.

### Vim

Сначала я создала рабочую директорию и тестовые файлы, с которыми дальше
работала в vim и mc:

mkdir ~/vim_mc_practice
cd ~/vim_mc_practice

echo "Line 1: Hello World" > test1.txt
echo "Line 2: Vim is awesome" > test2.txt
echo "Line 3: MC is powerful" > test3.txt
echo "number: 42" > numbers.txt
echo "number: 100" >> numbers.txt

ls -la
cat numbers.txt


Дальше я открыла файл `numbers.txt` в vim:

cd ~/vim_mc_practice
vim numbers.txt

Внутри vim я:
- включила нумерацию строк командой `:set nu`;
- потренировалась в навигации (`gg`, `G`, `0`, `$`, поиск `/number`);
- попробовала базовые команды редактирования: `dd`, `yy`, `p`, `u`, `Ctrl+r`;
- сохранила изменения и вышла с помощью `:wq`.


Для демонстрации продвинутого приёма я создала скрипт `script.sh`:

cat > script.sh << 'EOF'
#!/bin/bash
echo "Запуск сервиса"
systemctl start nginx
echo "Проверка статуса"
systemctl status nginx
echo "Сервис запущен"
EOF
vim script.sh

Дальше я сделала следующее:
1. Включила номера строк: `:set nu`.
2. Перешла на первую строку блока, который хочу закомментировать (например, `3gg`).
3. Нажала `V`, чтобы войти в режим выделения строк.
4. Нажимала `j`, чтобы выделить ещё несколько строк вниз.
5. Нажала `:` — внизу появилось `:'<,'>`.
6. Ввела команду `s/^/# /` и нажала Enter — у всех выделенных строк появился `#` в начале.

Так я разом закомментировала несколько строк:
"# echo "Запуск сервиса"
"# systemctl start nginx
"# echo "Проверка статуса"

Затем я удалила все закомментированные строки одной командой:

:g/^#/d

После этого все строки, начинающиеся с `#`, исчезли, а остальной код остался.

Этот приём показывает, как с помощью режима `V` и команд `s/^/# /` и `:g/^#/d`
можно очень быстро комментировать и убирать целые блоки кода.

### mc 

Затем я потренировалась работать с файловым менеджером mc:

cd ~/vim_mc_practice
mc

В mc я:
- переключалась между панелями клавишей `Tab`;
- просматривала файлы через `F3`;
- редактировала файлы во встроенном редакторе через `F4`;
- создала директорию `backup` с помощью `F7`;
- скопировала `test1.txt` в директорию `backup` с помощью `F5`;
- включила показ скрытых файлов сочетанием `Alt+.`.

После этого я убедилась, что в папке `backup` появился скопированный файл.

Чтобы зафиксировать весь процесс, я использовала asciinema.

Сначала я установила его и авторизовалась:

sudo dnf install asciinema        
asciinema auth                  

После этого я начала запись, находясь в директории с практикой:

cd ~/vim_mc_practice
asciinema rec --idle-time-limit 3 lab1_vim_mc.cast

Флаг `--idle-time-limit 3` позволяет автоматически обрезать длинные паузы,
чтобы запись получилась более компактной.

Всё, что я делала, автоматически
записывалось в файл `lab1_vim_mc.cast`. Когда все задания были выполнены,
я завершила запись командой `exit`

Потом я просмотрела запись:

asciinema play lab1_vim_mc.cast

И загрузила её на сайт, чтобы получить ссылку:

asciinema upload lab1_vim_mc.cast

### Анализ nginx access.log — топ‑10 IP

Я использовала подготовленный скрипт, который
создаёт большой лог nginx:

sudo bash 01_nginx_log_challenge.sh

После запуска в каталоге `/var/log/nginx/` появились файлы:

- `access.log` — обычный лог nginx на ~1M строк;
- `access_json.log` — тот же лог в JSON‑формате.

Чтобы найти 10 IP‑адресов с наибольшим количеством запросов в `access.log`,
я использовала утилиты:

awk '{print $1}' /var/log/nginx/access.log \
  | sort \
  | uniq -c \
  | sort -nr \
  | head -10

Здесь:
- `awk '{print $1}'` берёт первый столбец (IP‑адрес);
- `sort` сортирует IP, чтобы одинаковые оказались рядом;
- `uniq -c` считает количество запросов от каждого IP;
- `sort -nr` сортирует по числу запросов по убыванию;
- `head -10` оставляет только первые 10 строк (топ‑10).

Для файла `access_json.log`, где каждая строка это JSON с полем `"remote_addr"`,
я сделала аналогичное решение
Я извлекла IP‑адреса и посчитала топ‑10:

grep -o '"remote_addr": "[^"]*"' /var/log/nginx/access_json.log \
  | awk -F'"' '{print $4}' \
  | sort \
  | uniq -c \
  | sort -nr \
  | head -10

На выходе я получила такой же формат: слева количество запросов, справа IP‑адрес.
